# 军棋翻翻棋 Demo

基于 Rust 的双人在线军棋翻棋游戏，采用客户端-服务器架构，使用 Bevy 引擎渲染棋盘。

## 玩法简介

- 每方 25 枚棋子，开局全部暗置（背面朝上）
- 点击己方暗棋可以翻开；只有翻开的棋子才能移动或攻击
- 双方轮流操作：翻开、移动或攻击
- 吃掉对方军旗即获胜

### 棋子军衔（从低到高）

工兵 → 排长 → 连长 → 营长 → 团长 → 旅长 → 师长 → 军长 → 司令

特殊棋子：军旗（被吃即输）、地雷（只有工兵能排）、炸弹（同归于尽）

### 棋盘

12 行 × 5 列，包含三种地形：

- **道路** — 普通格子，每次只能走一步
- **铁路** — 可以直线快速移动，工兵可在铁路网上任意转弯
- **行营** — 安全区，营内棋子不可被攻击

## 技术栈

| 层     | 技术                |
| ------ | ------------------- |
| 语言   | Rust (edition 2024) |
| 客户端 | Bevy 0.18           |
| 网络   | 原生 TCP            |
| 序列化 | 自定义文本协议      |

## 构建与运行

需要 Rust 1.87+ 工具链。

```bash
# 构建
cargo build

# 启动服务器（监听 127.0.0.1:7878）
cargo run --bin junqi_server

# 启动客户端（两个终端分别运行）
cargo run --bin junqi_client
```

需要两个客户端先后连接（先连的为红方，后连的为蓝方），游戏才会开始。

## 项目结构

```
Cargo.toml                  # workspace 根
assets/fonts/simhei.ttf     # 中文字体
crates/
├── client/src/main.rs      # Bevy 客户端：渲染、输入、网络
├── server/src/main.rs      # TCP 服务器：游戏循环、状态管理
└── shared/src/             # 共享逻辑
    ├── types.rs            # 核心类型定义
    ├── board.rs            # 棋盘与地形
    ├── rules.rs            # 移动规则与战斗结算
    └── protocol.rs         # 网络协议编解码
```
